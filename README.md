This repository contains case studies and simulation notebooks for DASA (Dimensional Analysis for Software Architecture).

- The case studies focus on Self-Adaptive Systems/software, where dimensional analysis is particularly useful for quantifying trade-offs.
- Each case study has its own folder for data, results, images, and reports.
- Jupyter notebooks for each case study are located at the top level for easy access.
- See `SUMMARY.md` on each case study for a detailed overview of the context, data, and results.

## Repository Structure Summary

This repository contains two main case studies, each with their own data, results, images, and reports. Jupyter notebooks are located at the top level for easy access.

### About DASA

DASA stands for **Dimensional Analysis for Software Architecture**. The case studies in this repository are specifically focused on Self-Adaptive Systems/software, where dimensional analysis is particularly useful for quantifying trade-offs in system design and adaptation.

### Case Studies

- **CS-01 - E-Health - TAS**

  - TAS stands for **Tele Assistance System**
  - Tele Assistance System (TAS) case study
  - Input data, simulation results, images, and HTML reports are organized in dedicated folders
- **CS-02 - Smart City - SASR**

  - Smart City Self-Adaptable Service Index case study
  - Input data, simulation results, images, and HTML reports are organized in dedicated folders
