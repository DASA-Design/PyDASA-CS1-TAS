# Table Profile

## Cost QoS


| Column #           | Name                   | Description                                                                                                                                                                                                                                                                                            |
| ------------------ | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Column 1** | `Request ID`         | Sequential identifier for each service request (1, 2, 3, etc.)                                                                                                                                                                                                                                         |
| **Column 2** | `Service Name`       | The specific service invoked in the workflow:`<br>`- `MedicalService3` - Medical monitoring service`<br>`- `DrugService` - Medication management service`<br>`- `AlarmService3` - Alarm notification service`<br>`- `AssistanceService` - Final coordination service (end of workflow) |
| **Column 3** | `Success Status`     | Boolean indicating if the service invocation succeeded (`true` in all cases shown)                                                                                                                                                                                                                   |
| **Column 4** | `Service Cost`       | The**cost** associated with that specific service invocation (constant per service type:`<br>`- MedicalService3: 2.15`<br>`- DrugService: 5.0`<br>`- AlarmService3: 2.0`<br>`- AssistanceService: varies (7.15, 4.15, or 2.0 depending on workflow)                                      |
| **Column 5** | `Arrival Timestamp`  | **Absolute timestamp** (in seconds) when the service request arrived at that specific service (from simulation start)                                                                                                                                                                            |
| **Column 6** | `Response Time`      | **Actual response time** (in seconds) experienced by that service invocation - this is the waiting + processing time                                                                                                                                                                             |
| **Column 7** | `Total Request Cost` | **Cumulative cost** for the entire request workflow (only appears on the final `AssistanceService` row)                                                                                                                                                                                        |

## Prefered QoS

| Column #           | Name                   | Description                                                                                                                                                                                                                                                                                                                          |
| ------------------ | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Column 1** | `Request ID`         | Sequential identifier for each service request (1, 2, 3, etc.). Each request may invoke multiple services in a workflow.                                                                                                                                                                                                             |
| **Column 2** | `Service Name`       | The specific service invoked in the workflow:`<br>`- `MedicalService1/2/3` - Different medical monitoring services`<br>`- `AlarmService1/2/3` - Different alarm notification services`<br>`- `DrugService` - Medication management service`<br>`- `AssistanceService` - Final coordination service (end of workflow) |
| **Column 3** | `Success Status`     | Boolean indicating if the service invocation succeeded (`true` in all cases shown)                                                                                                                                                                                                                                                 |
| **Column 4** | `Service Cost`       | The**cost** associated with that specific service invocation (constant per service type):`<br>`- `MedicalService3`: 2.15`<br>`- `AlarmService3`: 2.0`<br>`- `DrugService`: 5.0`<br>`- `AssistanceService`: varies based on workflow path                                                                       |
| **Column 5** | `Arrival Timestamp`  | **Absolute timestamp** (in seconds) when the service request arrived at that specific service (from simulation start). This increases as the request progresses through the workflow.                                                                                                                                          |
| **Column 6** | `Response Time`      | **Actual response time** (in seconds) experienced by that service invocation - this includes waiting time in queue + processing time. This is a  **key performance metric** .                                                                                                                                            |
| **Column 7** | `Total Request Cost` | **Cumulative cost** for the entire request workflow. This **only appears on the final row** (AssistanceService) of each request and sums up all service costs in the workflow.                                                                                                                                           |

## **Reliability QoS**


| Column #           | Name                   | Description                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------ | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Column 1** | `Request ID`         | Sequential identifier for each service request (1, 2, 3, etc.). Each request may invoke 2-3 services in a workflow.                                                                                                                                                                                                                                                                            |
| **Column 2** | `Service Name`       | The specific service invoked in the workflow:`<br>`- `MedicalService2` - Medical monitoring service (Reliability QoS variant, cost: 14.0)`<br>`- `AlarmService2` - Alarm notification service (Reliability QoS variant, cost: 12.0)`<br>`- `DrugService` - Medication management service (cost: 5.0)`<br>`- `AssistanceService` - Final coordination service (end of workflow) |
| **Column 3** | `Success Status`     | Boolean indicating if the service invocation succeeded (`true` in all cases - this is expected for Reliability QoS)                                                                                                                                                                                                                                                                          |
| **Column 4** | `Service Cost`       | The**cost** associated with that specific service invocation:`<br>`- `MedicalService2`: 14.0 (higher cost for more reliable service)`<br>`- `AlarmService2`: 12.0 (higher cost for more reliable service)`<br>`- `DrugService`: 5.0`<br>`- `AssistanceService`: total workflow cost (19.0, 12.0, or 26.0)                                                                |
| **Column 5** | `Arrival Timestamp`  | **Absolute timestamp** (in seconds) when the service request arrived at that specific service (from simulation start at ~118738 seconds). Increases as the request progresses through the workflow.                                                                                                                                                                                      |
| **Column 6** | `Response Time`      | **Actual response time** (in seconds) experienced by that service invocation - includes waiting time in queue + processing time. This is the  **key performance metric** .                                                                                                                                                                                                         |
| **Column 7** | `Total Request Cost` | **Cumulative cost** for the entire request workflow. This **only appears on the final row** (AssistanceService) of each request.                                                                                                                                                                                                                                                   |
